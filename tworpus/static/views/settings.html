Tweet-Cache

  • When a corpus is created the fetched tweets are additionally saved on your PC in a separate folder
  • As a result of this the prospective corpus creation processes are sped up, if the files were fetched
  • This needs extra memory space and if the cache folder is getting too large, you can delete it

Tweets per XML

  • Handling large XML files need a certain amount of main memory
  • To accomplish corpus creation for older PCs you can set the tweet number per file
  • The default value is set to 10000, that means that one created XML file contains 10000 tweets at most
  • If your corpus contains more tweets, several XML files are generated
  • If you know what you do, you can increase this value
